# Code of Conduct

Bitcoin for the Arts Research is a public, citable research program. The way we work together is part of the product. This Code of Conduct applies to anyone interacting with this repository — issues, pull requests, discussions, comments, and any communication channel associated with the project.

---

## Our standard

We aim for the working culture of a serious research institution: rigorous, candid, and impersonal in dispute.

Contributors and maintainers commit to:

- **Precision** — argue with sources, not assertions
- **Respect** — disagree without disparaging the person
- **Transparency** — surface conflicts of interest, funding relationships, and prior positions where relevant
- **Patience** — review takes time; primary-source verification takes time; this is the standard, not a delay
- **Public correction** — when we are wrong, we say so plainly and fix it in the record

---

## Unacceptable behavior

- Personal attacks, harassment, intimidation, or discrimination on any basis
- Sustained disruption of issues, pull requests, or discussions
- Publishing private information of others without explicit permission
- Submitting claims known to be unsourced, fabricated, or AI-hallucinated as if they were verified evidence
- Attempting to push the project outside its [stated ethos](ETHOS.md) — e.g. via repeated PRs advocating for altcoins, NFTs, ordinals, or non-monetary uses of the Bitcoin network
- Any conduct that would be unwelcome in a professional research environment

---

## Enforcement

Maintainers may, at their discretion:

- Edit, hide, or remove comments, commits, code, issues, or pull requests that violate this Code of Conduct
- Lock threads
- Block contributors temporarily or permanently from the repository

Concerns may be reported confidentially to `conduct@bitcoinforthearts.org` (or `hello@bitcoinforthearts.org` if the dedicated address is unavailable). Reports are reviewed promptly and confidentially. Reporters are not retaliated against.

---

## Scope

This Code of Conduct applies in all repository spaces (issues, pull requests, discussions, code review) and in public spaces when an individual is representing the project — for example, citing themselves as a Bitcoin for the Arts Research contributor in a public talk, article, or social-media post.
