# Newsletters

Archive of newsletters published by **Bitcoin for the Arts Research** and Bitcoin For The Arts, Inc.

BFTA's primary newsletter channels:

- **Beehiiv** — operational and programmatic updates
- **Substack** — long-form editorial and research framings
- **Site newsletter** — research-focused mailings linked from [bitcoinforthearts.org](https://bitcoinforthearts.org)

This folder mirrors the **research-relevant** issues — those that report a fact, summarize new findings, link to a report, or document a development in the funding landscape. Operational and fundraising-only mailings are kept in the main BFTA repository and not duplicated here.

## Layout

Each issue lives in its own file or folder named with the publication date:

```
newsletters/
├── 2026-04-27-vegas-conference.md
├── 2026-05-04-issue-6.md
└── ...
```

Each file begins with a short front matter block:

```
---
title: Issue title
channel: beehiiv | substack | site
published: YYYY-MM-DD
url: https://...
---
```

Followed by the text of the issue with **every factual claim sourced** per [`../CITATIONS.md`](../CITATIONS.md).

## Provenance

Beehiiv drafts and other newsletter source files in the main BFTA repository live at:

- `Bitcoin-For-The-Arts/Bitcoin-For-The-Arts/docs/substack-strategy/beehiiv-newsletters/`
- `Bitcoin-For-The-Arts/Bitcoin-For-The-Arts/docs/newsletter/`

When a newsletter issue is research-relevant, the editorial draft from those folders is reviewed against the citation standard and mirrored here as the canonical archive copy.

## Citation standard

Newsletters follow the same citation discipline as reports and articles — see [`../METHODOLOGY.md`](../METHODOLOGY.md). Short format is fine. Unsourced claims are not.
